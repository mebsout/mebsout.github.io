Command line interface


usage: proofmgr <action> [options]

Actions:
  -add-prover             <name> <path> <options> <version> adds a prover to the database
  -add-bench -c           <cat> [-n <name>] <path> adds a benchmark to the database
  -remove-prover          <id> removes a prover from the database
  -remove-bench           <id> removes a benchmark from the database
  -remove-result          <idprover> <idbench> removes a result from the database
  -remove-results         <idprover> removes all results for one prover from the database
  -show-provers           shows the provers in the database
  -show-benchs            shows the benchmarks in the database
  -show-categories        shows existing benchmark categories
  -show-result            <idprover> <idbench> show one result
  -show-results           show the results that are in the database
  -compare                <id1> <id2> compare the results of prover with id <id1> and with id <id2>
  -export-all file.csv    export all results for all provers in file.csv
  -export-html file.html  export all results for all provers in HTML format for later viewing
  -report name            file.html generate a report for prover <name> in HTML format for
                          later viewing
  -run                    (-i <id> |-n <name>) ... runs the prover on the selected benchmarks 
                          (on all of them if none is selected)

Options:
  -c <cat>                sets the category of benchmaks to use to <cat>
  -category <cat>         sets the category of benchmaks to use to <cat>
  -i <id>                 sets the id to use
  -e                      show errors on results
  -local <dir>            set working directory on local machine or nfs system
  -remote <user@host:dir> set working directory on remote machine
  -distributed            perform computations on a set of machines
  -cores <n>              set number of cores to be used for local computations (default = number of cores on the machine)
  -port <port>            port to use for distributed computations 
                                (default 51001)
  -n <name>               the name argument of the action to be taken
  -name <name>            the name argument of the action to be taken
  -nocolor                do not use ANSI colors
  -t <seconds>            the timeout in seconds
  -timeout <seconds>      the timeout in seconds
  -verbose                makes proofmgr be verbose
  -help                   Display this list of options
  --help                  Display this list of options
